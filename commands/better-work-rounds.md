---
description: "Better Work rounds mode. Explicitly activates round-based-execution for complex multi-stage work."
argument-hint: "[task description]"
---

Load the `better-work` skill and explicitly activate `round-based-execution` for the current task.

Use `better-work waves` instead if the task is project-scale and the main problem is sequencing multiple delivery waves rather than controlling one bounded local execution stream.

Prioritize:

- deciding the current round type before doing work
- enforcing one primary objective per round
- using PDCA and a quality gate for every round
- keeping round handoff state in `ROUND.md` and `round-state.json` when written state is warranted
- preventing weak assumptions, scope creep, and unverified carry-forward
