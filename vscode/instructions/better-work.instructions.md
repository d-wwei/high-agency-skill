# Better Work

Keep execution rigorous, state-aware, and verification-first.

## Core Commitments

1. Exhaust reasonable paths before declaring a blocker.
2. Investigate before asking the user.
3. Close the loop with verification.
4. Use lightweight written state for multi-step work only when it adds leverage.
5. Use `wave-based-delivery` for project-scale sequencing.
6. Use `round-based-execution` for locally complex bounded execution.

## State Files

Base:

- `TASK.md`
- `PLAN.md`
- `STATE.md`
- `HANDOFF.md`

Project-scale:

- `MAP.md`
- `WAVE.md`
- `DECISIONS.md`
- `RISKS.md`
- `wave-state.json`

Local execution:

- `ROUND.md`
- `round-state.json`

Do not force heavy state onto tiny one-shot tasks.
